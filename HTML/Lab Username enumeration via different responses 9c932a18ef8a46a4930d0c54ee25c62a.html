Lab: Username enumeration via different responses

This lab is vulnerable to username enumeration and password brute-force attacks. It has an account with a predictable username and password, which can be found in the following wordlists:

usernames :

passwords:

Solution:

Using hyrda we will be finding the username and password

  1. Create a text file and copy and paste the usernames list that has been give. Save the usernames to a .txt file named users.txt
  1. Create another text file and copy and paste the passwords list that has been give. Save the passwords to a .txt file names passwords.txt
  1. Click the My account link in the top right
  1. Enter a random username and password to see what the error message is (We will need it for later).
  1. We then do this again but first right click the page and click Inspect to open developer tools.
  1. Your screen will look something like this.
  1. click on the Network tab
  1. Next click the Log in button. The Network tab will look like this after.
  1. In the Network tab click the item that called login.
  1. Click the Payload tab.
  1. Then click view source.
  1. We will then get back something that looks like this.
  1. Save this it will be useful for hyrda later. i.e username=Random+user+name&password=run
  1. Take the url for the page and save it. i.e 0a4e006e036495a58663d50400d90023.web-security-academy.net/login
  1. Open ternimal and go to where you have saved uses.txt and password.txt
  1. We are then going to run hydra -L users.txt -p <random letter or number> <URL> https-post-form "/login:username=^USER^&password=^PASS^:Invalid username"i.e hydra -L users.txt -p p 0a4e006e036495a58663d50400d90023.web-security-academy.net https-post-form "/login:username=^USER^&password=^PASS^:Invalid username"
  1. We will end up with getting back some information that will be the username.
  1. Once we have the username we now need to go back the site and see what the error is for an invade password.
  1. We use hydra again but change the two things. First the hydra -L users.txt will now be hydra -l <usersname> . Second change Invalid username to Incorrect password at the end of the hydra command . i.e "/login:username=^USER^&password=^PASS^:Invalid username" will now be "/login:username=^USER^&password=^PASS^:Incorrect password"
  1. The code for the terminal should be hydra -l <USERNAME> -P passwords.txt <URL> https-post-form "/login:username=^USER^&password=^PASS^:Incorrect password" i.e hydra -l user -P passwords.txt 0a4e006e036495a58663d50400d90023.web-security-academy.net https-post-form "/login:username=^USER^&password=^PASS^:Incorrect password"
  1. There should now be only one username and password
  1. Put in the username and password and it should be done.
